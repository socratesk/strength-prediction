Exploratory Data Analysis

Exploratory Data Analysis (EDA) is performed to understand the data, to get more insight to perform better feature enigneering, and to clean-up/ tidy the dataset. Before performing feature engineering and model generation, it is necessary to understand
  • whether it is a right data
  • whether there is enough data
  • the spread of the data
  • the number of missing values
  • the correlation between features
  • whether there is a data leakage
  • the feature contribution/ importance
  • whether there are any outliers
  • whether there are duplicate instances (rows)
  • whether there are duplicate features (columns) or feature with same value
